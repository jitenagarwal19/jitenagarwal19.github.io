Unlocking Potential of Terminal One Command at a Time.

I am not a pro user of terminal, but recently I started working on technologies which forced me to use CLI(Command Line Interface) for tasks like building source, running server, deploying the code etc. Generally when you use GUI while you figure out a workflow which is most optimized for you this might include knowing all the keyboard shortcuts, optimizing the Editor’s layout according to you, creating shortcuts on your Desktop and many more which I cannot think of right now. But when you are working on terminal your choices are limited by how you use commands provided and remembering them is a daunting task. As a programmer I feel the power of making my computer work the way I want, so I had to figure out a way to make Terminal work for me to make my workflow optimized.

Aliasing

Word Alias has many meanings if you will google it but the one that suits here is an alternate name for someone or something. In terminal also we can pretty much alias anything we want, meaning we can rename any complicated command in our own way.

How this is helpful?-You may ask Let me show you how -

Let’s say everytime I want to work on my personal website I have to write following commands in the terminal -

  • $cd ~/Documents/myProjects/myWebsite to change my working directory
  • $code . - To open VS Code (Editor)
  • $hugo serve - To start my local server

After Aliasing I just write $mySite_run. This single commands executes all three commands.

Enough jibber-jabber show me how to do it Look for ~/.bash_profile, ~/.bash_login or ~/.profile which are files which are executed whenever you open the terminal, we will add our alias command here so that we can use them whenever terminal starts. If you weren’t able to find any of the aboove mentioned files type touch ~/.bash_profile to create new file. Once the file is created you can open it any editor of your choice and start editing it. For the example that I gave above alias command will look like this

alias mySite_run='cd ~/Documents/mySite/; code .;hugo server'

; in command helps to execute more than one commands in one go.

Other alias that I use are following - - alias poweroff='sudo /sbin/shutdown -h now' - for force mac shutdown - alias mysql='/usr/local/mysql-5.5.25-osx10.6-x86_64/bin/mysql -u root' - to start mysql installation

You can add as many aliases you want and optimize your daily workflow.

One quick pro tip

If you are like me and keep forgetting stuff I like to print my alias whenever I open my terminal using echo so lets I write

echo -e 'Use\nmySite_run-to edit website\namazon_project-to edit amazon_project'

which will give me output like this whenever I open the terminal which helps me keep track of my aliases
